Binary Numbers

Introduction

Terminology & Nomenclature

  • bias — The value of the 'k' in excess-k.
  • biased exponent — The exponent bit representation after the bias has been applied.
  • excess-k (also excess-N) — A scheme in which the minimum and maximum value of a system are shifted by adding a bias, such that the new minimum value is all zeroes and the new maximum values is all ones.
  • mantissa (also significand) — The 'fraction' portion of a number written in scientific notation.

Note: excess-k has a different range than two's complement. For example, the range of 8-bit excess-k is [-127, 128] while the range of two's complement is [-128, 127].

Unsigned Magnitude

One representation of zero

N-bit system Range of Values Size of Range
N [0, 2N-1] 2N
8 [0, 255] 256

Summary

Unigned magnitude is your plain old counting binary.

Writing an Unsigned Magnitude Number

  1. Divide the number by two to obtain an integer quotient and a remainder.
  2. Put the remainder in the next unoccupied position from smallest to largest.
  3. If the quotient is zero, stop. Else, repeat where the number is now the quotient.

Issues

There are no negative numbers!

Signed Magnitude

Two representations of zero

N-bit system Range of Values Size of Range
N [-2N-1+1, 2N-1-1] 2N-1
8 [-127, 127] 255

Summary

Signed magnitude builds on unsigned magnitude by placing a 0 or a 1 at the start of a number to represent positive or negative, respectively. The rest of the digits then signify the value or 'magnitude' of the number. Because computer architectures can only handle so many bits in a processor (the precision), the number is written as fixed-width. Thus leading zeroes are necessary to pad the number.

Writing a Signed Magnitude Number

  1. Check the number is in range for the number of bits you have to work with.
  2. Set aside the sign for a moment, and treat the number as positive.
  3. Get the value as you would in unsigned magnitude.
  4. Pad with leading zeroes as necessary.
  5. If positive, set the leading bit to zero. If negative, set the leading bit to one.

Issues

Arithmetic is difficult and there are two representations of zero.

Writing a One's Complement Number

  1. Check the number is in range for the number of bits you have to work with.
  2. Set aside the sign for a moment, and treat the number as positive.
  3. Get the value as you would in unsigned magnitude.
  4. Pad with leading zeroes as necessary.
  5. If positive, stop. If negative, flip all the bits.

Issues

Arithmetic almost works, but we have to check for both values of zero.

Two's Complement

One representation of zero

N-bit system Range of Values Size of Range
N [-2N-1, 2N-1-1] 2N
8 [-128, 127] 256

Summary

Two's complement is the most widespread number format. Addition 'simply' works. Subtraction is represented as addition: A-B → A+neg(B)

Writing a Two's Complement Number

  1. Check the number is in range for the number of bits you have to work with.
  2. Set aside the sign for a moment, and treat the number as positive.
  3. Get the value as you would in unsigned magnitude.
  4. Pad with leading zeroes as necessary.
  5. If positive, stop. If number, flip all the bits and add one.
