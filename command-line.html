The Command Line

The Shell and the Command Line

The terms 'shell', 'command line', 'command prompt' and others are colloquially interchangeable, but there is a difference. A shell is a program that acts as the intermediary between you and the operating system. The shell is separate from the interface used to accept input and convey output. For example, on macOS the default shell is bash and the default command line interface is Terminal. On Windows, the shell is Command Prompt (cmd.exe) and the command line interface is Win32 Console.

While graphical applications can be used to manage file transfers between your computer and Sparky, I recommend taking the opportunity now to familiarize yourself with the command line. As such, I will only cover command line access.

Linux and macOS users: you already have the tools necessary to interact with Sparky.

Windows users: At the time of writing, Windows is not natively capable of SSH on the command line. This may change with Windows PowerShell but alternatively I recommend you download and install Git Bash from https://git-scm.com/downloads.

Using Programs on the Command Line

Note: Your prompt text and color schemes may vary.

Note: The tilde is commonly used in UNIX systems as shorthand for the signed-in user's home folder. My home folder is /home/sean and a folder thus the folder /home/sean/Documents would be represented as ~/Documents

You can provide arguments to programs on the command line with the use of flags. Program flags often have both a short flag and long flag version. For example, look at the command line program ls (widely available on UNIX-derived systems). This program lists all the files in a directory. If we were to use ls in an empty directory, it would look like this:

sean@laptop [~/empty-dir] $ ls
sean@laptop [~/empty-dir] $

By default ls only shows non-hidden files. If we want to see all files we can provide the all flag:

sean@laptop [~/empty-dir] $ ls -a
.  ..
sean@laptop [~/empty-dir] $ ls --all
.  ..
sean@laptop [~/empty-dir] $

There are three things to appreciate about these results:

  1. The all flag has a short version (-a) and a long version (--all) and this is the case for many arguments for many programs.
  2. Every folder has at least two things in it: . and .. which are respectively references to the current folder and the folder one level up.
  3. Hidden files and folders in UNIX-derived systems begin with a dot which is why we were unable to see these before.
